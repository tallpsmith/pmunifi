# hatch-vcs Versioning Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace hardcoded version strings with git-tag-derived versioning via hatchling + hatch-vcs, so a single `git tag v0.X.0` drives PyPI, RPM, and DEB package versions.

**Architecture:** Switch build backend from setuptools to hatchling with hatch-vcs plugin. At build time, hatch-vcs reads the git tag and writes `_version.py`. The release workflow validates that the GitHub Release tag matches the built version, and injects the version into RPM spec and DEB changelog via sed before packaging.

**Tech Stack:** hatchling, hatch-vcs, GitHub Actions

Hatchling auto-includes package data by default, so no explicit `package-data` equivalent is needed. The `deploy/*` files inside `src/pcp_pmda_unifi/deploy/` will be included automatically.

### Task 5: Update release.yml with tag validation and version injection

**Files:**
- Modify: `.github/workflows/release.yml`

- [ ] **Step 1: Add tag validation step to the pypi job**

Add as the first step after checkout in the `pypi` job:

```yaml
      - name: Validate tag matches package version
        run: |
          TAG="${GITHUB_REF#refs/tags/v}"
          PKG_VERSION=$(python -c "
          import subprocess, re
          result = subprocess.run(['python', '-m', 'build', '--sdist', '--no-isolation'], capture_output=True)
          " || true)
          # Build first, then check
          pip install hatchling hatch-vcs
          PKG_VERSION=$(python -c "from importlib.metadata import version; print(version('pcp-pmda-unifi'))" 2>/dev/null || \
            python -c "import tomllib, subprocess; print(subprocess.check_output(['hatchling', 'version']).decode().strip())")
          echo "Git tag version: $TAG"
          echo "Package version: $PKG_VERSION"
          if [ "$TAG" != "$PKG_VERSION" ]; then
            echo "::error::Tag v$TAG does not match package version $PKG_VERSION"
            exit 1
          fi
```

Actually, a simpler approach — hatchling can report the version directly:

```yaml
      - name: Validate tag matches built version
        run: |
          pip install hatchling hatch-vcs
          TAG="${GITHUB_REF#refs/tags/v}"
          PKG_VERSION=$(hatchling version)
          echo "Git tag: v$TAG → Package version: $PKG_VERSION"
          [ "$TAG" = "$PKG_VERSION" ] || { echo "::error::Version mismatch!"; exit 1; }
```

- [ ] **Step 2: Update the pypi job build step**

Change:
```yaml
      - run: pip install build
      - run: python -m build
```
To:
```yaml
      - run: pip install build hatch-vcs
      - run: python -m build
```

(`hatchling` is pulled in as a build dep from pyproject.toml, but `hatch-vcs` needs to be available for the VCS version lookup.)

- [ ] **Step 3: Update the RPM job to inject version from git tag**

In the `rpm` job, after checkout, add version extraction and injection:

```yaml
      - name: Extract version from tag
        id: version
        run: echo "version=${GITHUB_REF#refs/tags/v}" >> "$GITHUB_OUTPUT"

      - name: Inject version into RPM spec
        run: sed -i "s/@@VERSION@@/${{ steps.version.outputs.version }}/g" packaging/rpm/pcp-pmda-unifi.spec
```

Update the build step to use hatchling:
```yaml
      - run: pip install build hatch-vcs
```

Update the rpmbuild source copy to use the version variable:
```yaml
      - name: Prepare rpmbuild tree
        run: |
          mkdir -p ~/rpmbuild/{SOURCES,SPECS,BUILD,RPMS,SRPMS}
          VERSION="${{ steps.version.outputs.version }}"
          cp dist/pcp_pmda_unifi-${VERSION}.tar.gz \
             ~/rpmbuild/SOURCES/pcp-pmda-unifi-${VERSION}.tar.gz
          cp packaging/rpm/pcp-pmda-unifi.spec ~/rpmbuild/SPECS/
```

(Removes the `tomllib` one-liner — the tag is now the single source of truth.)

- [ ] **Step 4: Update the DEB job to inject version from git tag**

In the `deb` job, after checkout, add:

```yaml
      - name: Extract version from tag
        id: version
        run: echo "version=${GITHUB_REF#refs/tags/v}" >> "$GITHUB_OUTPUT"
```

Update the changelog creation to use the extracted version:
```yaml
      - name: Prepare debian directory
        run: |
          cp packaging/deb/control debian/control
          cp packaging/deb/rules debian/rules
          cp packaging/deb/postinst debian/postinst 2>/dev/null || true
          chmod +x debian/rules
          VERSION="${{ steps.version.outputs.version }}"
          cat > debian/changelog << CHANGELOG
          pcp-pmda-unifi (${VERSION}-1) unstable; urgency=low

            * Release ${VERSION}

           -- pmdaunifi contributors <pmdaunifi@users.noreply.github.com>  $(date -R)
          CHANGELOG
          echo "debian/pmdaunifi.1" > debian/pcp-pmda-unifi.manpages
```

Add build deps:
```yaml
      - run: pip install build hatch-vcs
```
